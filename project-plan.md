# GyakutenMaker - 逆转裁判同人游戏制作器

## 项目愿景

制作一个**可视化、无需编码**的前端工具，让逆转裁判玩家能够制作自己的逆转裁判同人游戏。

类似视频/音频编辑软件的工作方式：通过切换不同环节系统、添加素材等方式，在可视化页面中完成整个逆转裁判剧情流程的制作。

## 为什么不用 Ren'Py

Ren'Py 是优秀的通用视觉小说引擎，但逆转裁判系列有许多 Ren'Py 无法在前端无代码实现的特有机制：

### 文本表现层
- 角色说话时有**嘟嘟声**，音效随字幕出现速度变化
- 同一句话中，不同字的出现速度可以不同（非统一速度浮现）
- "异议あり！""待った！""くらえ！"等大字冲击动画，带缩放和震动，画面闪白
- 关键台词会**突然停顿**再继续，制造悬念感
- 角色情绪激动时文字会**震动/放大**

### 法庭/询问系统
- 证言逐条展示，玩家可以**威慑**（追问细节）或**举证**（指出矛盾），UI 和反馈完全不同
- 举证错误时**血槽扣减**，扣完 Game Over，血槽有独特动画和音效
- 威慑时角色可能**追加新证言**，证言列表动态变化
- 同一证人不同证言对应的**角色立绘/动画不同**
- **叮！**音效，每句证言之间有节奏感的停顿

### 搜证环节
- 场景可**自由调查**，点击特定区域触发对话或获取证物
- 与角色对话时可选择**出示证物**，不同证物触发不同反应
- **心锁系统**（心灵枷锁）：需按正确顺序出示正确证物才能解开，每次失败扣血

### 演出动画
- 拍桌子（检/辩双方各有不同桌子动画）
- 手指指向对方（经典"异议"姿势）
- 角色**崩溃**动画（被揭穿后流汗、后退、头发散乱等）
- 屏幕震动、黑白闪、慢镜头等转场效果

### 系统层面
- 证言回溯：可随时重新查看已解锁证言
- 证物组合：某些矛盾需同时出示两件证物
- 心灵枷锁、情绪矩阵、看穿系统等特殊机制

## 开源定位

- 目标用户：逆转裁判玩家（非程序员）
- 开源到 GitHub，社区驱动
- 需要极低的使用门槛

## 技术栈

- 前端/编辑器：React + TypeScript + Vite
- 数据格式：JSON + JSON Schema（可校验）
- 部署：GitHub Pages（播放器）+ Tauri（桌面版，可选）
- 包管理：Monorepo（引擎核心和编辑器分开）

## 开发路线

### 阶段 0 — 最小可用原型
- 用 JSON 定义场景数据格式
- 实现基础对话系统 + 法庭举证环节
- 简单 Web 播放器，加载 JSON 并播放
- 放到 GitHub，写好 README

### 阶段 1 — 社区反馈 + 引擎完善
- 根据反馈调整数据格式
- 补充搜证环节、心锁系统等
- 加入基础演出（拍桌、震动、文字效果）
- 提供示例关卡

### 阶段 2 — 可视化编辑器
- 基于已定型的数据格式做前端编辑器
- 时间轴/流程图式场景编辑
- 素材管理（角色立绘、音效、BGM）
- 实时预览

### 阶段 3 — 完善和推广
- 模板系统
- 文档和教程
- 社区素材分享机制

## 数据格式设计原则

- 人类可读可写
- 版本兼容（加 version 字段）
- 自包含（一个关卡文件夹包含所有素材，方便分享）
